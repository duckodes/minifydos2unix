# 快速行尾符號轉換工具

## 與[官方](https://github.com/tizenorg/platform.upstream.dos2unix) `dos2unix` 的差異

| 功能面向 | [官方](https://github.com/tizenorg/platform.upstream.dos2unix) `dos2unix` | `minifydos2unix` |
|----------|----------------|------------------|
| **主要用途** | DOS/Windows → Unix 文字檔轉換 | 專注於 CRLF ↔ LF 換行符號轉換 |
| **編碼處理** | 支援多種 DOS code page (CP437、CP850、CP1252 等) → ISO-8859-1 | 不處理編碼，保留原始位元組 |
| **二進位檔檢查** | 會檢測控制字元，避免誤轉二進位檔 | 不檢查，單純替換換行符號 |
| **複雜度** | 功能完整，程式碼龐大複雜 | 精簡、輕量，邏輯單純 |
| **速度** | 多步驟處理，稍微重 | 更快，因為只做字串替換 |
| **現代環境適用性** | 為舊系統相容性設計，UTF-8 環境下編碼轉換幾乎用不到 | 完全符合現代 UTF-8 環境需求 |
| **定位** | 歷史包袱 + 全功能工具 | 精簡版，專注於「行尾符號轉換」 |

C# 撰寫的輕量化工具，用於在文字檔中轉換行尾符號格式：  
- LF (Unix/Mac)  
- CRLF (Windows)  

本工具僅進行行尾符號轉換，不修改檔案編碼

## 功能
- LF/CRLF 互轉
- 保留原始檔案編碼 (如 UTF-8、UTF-16)

## 安裝
1. 執行安裝程式並完成設定
2. 安裝完成後，可從桌面捷徑或開始功能表啟動

## 使用方式
1. 開啟 **行尾符號轉換工具**
2. 點選 **選擇檔案**，挑選欲轉換的文字檔
3. 選擇目標行尾格式 (LF 或 CRLF)
4. 即可輸出成目標行尾格式

## 注意事項
- 僅適用於文字檔，請勿用於二進位檔。
- 不更改檔案編碼，只處理行尾符號。
